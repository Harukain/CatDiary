# 猫伴日记商店隐私披露基线

更新时间：2026-07-12

本文件是 App Store Connect、Google Play Data safety 和正式隐私政策的工程事实底稿，不替代法律文本。提交前必须由产品所有者补充运营主体、联系邮箱、隐私政策 URL、用户协议 URL、数据存储地域和第三方处理者清单，并由合规人员复核。

## 1. 数据类型与用途

| 数据                                                           | 是否关联账号 | 是否用于追踪 | 用途                           | 保存与删除                                                          |
| -------------------------------------------------------------- | ------------ | ------------ | ------------------------------ | ------------------------------------------------------------------- |
| 手机号                                                         | 是           | 否           | 验证码登录、家庭邀请、账号安全 | 服务端加密保存并使用 HMAC 索引；注销到期后匿名化                    |
| 用户 ID、家庭/成员关系                                         | 是           | 否           | 登录会话、家庭协作和权限隔离   | 随账号和法定留存策略删除或匿名化                                    |
| 设备推送 Token                                                 | 是           | 否           | 发送用户主动开启的照顾提醒     | 用户关闭、设备失效或会话撤销后停用/删除                             |
| 猫咪档案、任务、饮食、排便、呕吐、体重、疫苗、驱虫、用药和备注 | 是           | 否           | 用户请求的养猫记录与提醒功能   | 用户可修改/删除；本机记录缓存 90 天、任务缓存 7 天                  |
| 猫咪照片及备注                                                 | 是           | 否           | 家庭相册、猫咪头像和就医资料   | 私有对象存储；软删除后按保留策略清理；本机孤儿副本 24 小时回收      |
| 导出文件                                                       | 是           | 否           | 响应用户的数据导出请求         | 文件 7 天过期，下载链接 10 分钟失效                                 |
| 请求 ID、低基数路由、错误码和性能指标                          | 原则上否     | 否           | 安全、故障诊断和服务稳定性     | 日志禁止记录验证码、Token、完整手机号、Webhook、签名 URL 和健康备注 |

猫咪健康记录不是人的 Apple Health/健康与健身数据，不接入 HealthKit，也不得在商店问卷中误报为人体健康数据。它仍属于用户内容和敏感业务数据，隐私政策应单独清楚说明。

## 2. iOS Privacy Manifest

App 自身声明不追踪、不包含追踪域名；收集的数据只用于 App Functionality，并与账号关联。声明的数据类型为手机号、用户 ID、设备 ID（推送 Token）、照片/视频和其他用户内容。

Required Reason API 条目来自当前安装的 Expo SDK 57 与 React Native 0.86 各自 `PrivacyInfo.xcprivacy` 的并集：

| API 类别         | 原因代码                     |
| ---------------- | ---------------------------- |
| File Timestamp   | `0A2A.1`、`3B52.1`、`C617.1` |
| Disk Space       | `85F4.1`、`E174.1`           |
| System Boot Time | `35F9.1`                     |
| User Defaults    | `CA92.1`                     |

升级 Expo、React Native 或加入原生 SDK 时，必须重新扫描依赖 Privacy Manifest，并更新 `app.config.ts` 与自动门禁。Apple 对 Required Reason API 列表会持续调整，TestFlight 上传后仍需检查 App Store Connect 邮件和隐私报告。

## 3. Google Play Data safety

建议按“由开发者收集、传输中加密、可请求删除、不用于广告或跨应用追踪”填报手机号、用户标识、照片、其他用户生成内容和 App 功能交互。若腾讯云短信、COS、Expo Push 或后续监控服务作为服务提供商处理数据，必须在正式隐私政策列明处理目的、数据类别、地域和退出/删除方式。

当前 App 不申请定位、联系人、麦克风或广告 ID 权限；Android 系统备份关闭。Google Play 表单应以最终 AAB 的 Data safety/SDK 报告为准，不能只根据本文件勾选。

## 4. 提交前待确认

- 运营主体法定名称与联系地址
- 客服邮箱、隐私邮箱和账号删除联系渠道
- 隐私政策与用户协议的 HTTPS URL
- 腾讯云 PostgreSQL、Redis、COS、短信的实际地域
- Expo Push 及腾讯云作为处理者的正式披露措辞
- 数据保留周期是否满足适用地区法律和业务承诺
- App Store Connect 与 Google Play 问卷截图由两人复核并归档

Preview/Production 构建通过 `EXPO_PUBLIC_PRIVACY_POLICY_URL` 与 `EXPO_PUBLIC_TERMS_URL` 注入正式地址；构建配置会拒绝缺失、HTTP、本机地址、带凭据、查询参数或 fragment 的 URL。登录页和登录后的设置页均提供入口，但正式内容及法律效力仍由运营主体负责确认。
