- M1 设备会话：会话列表、单设备撤销、当前设备退出、全部设备退出。
- M1 家庭基础：创建、列表、详情、乐观锁更新。
- M1 家庭协作：手机号绑定邀请、接受邀请、成员列表、角色调整、移除、主动离开、最后管理员保护。
- M1 猫咪档案基础：创建、列表、详情、乐观锁更新、软删除、每家庭最多 5 只。
- 安全边界：`X-Family-Id` 上下文、active membership 校验、管理员写权限 Guard、家庭资源隔离。
- 审计：邀请、接受、角色调整、成员移除和离开操作写入不可变审计日志。
- App 登录与建档：手机号与验证码、60 秒重发、Refresh Token 安全存储与启动恢复、创建家庭、添加猫咪、退出登录。
- App 路由：正式迁移到 Expo Router，拆分认证、建档、五 Tab、家庭成员、猫咪列表/详情和邀请深链。
- App 导航：实现已确认的 `首页｜任务｜＋｜记录｜我的`，相册未占一级 Tab。
- App 家庭与档案：家庭切换、成员列表/邀请/角色调整/移除、猫咪列表与名称编辑。
- App 会话可靠性：统一 401 自动续期、Refresh Token 单航班并发锁、轮换状态更新和失效回登录。
- App 猫咪完整资料：名字、性别、生日、品种、绝育、芯片号和带后果说明的软删除。
- M2 计划基础：计划 CRUD、暂停/恢复、版本锁、重复规则、IANA 时区与未来 7 天任务生成。
- M2 任务基础：今天/即将/逾期/完成列表，负责人、完成、跳过、撤销，完成时原子生成记录。
- M2 医疗保护：疫苗、驱虫和用药任务完成前强制确认；重复完成和并发版本冲突受保护。
- M2 App：新建疫苗/驱虫/用药/铲屎计划，选择猫咪、时间与单次/日/周/月重复规则。
- M2 App 任务：今天/即将/逾期/完成四个视图，完成、补完成、跳过、医疗确认和撤销交互。
- M2 Worker：每 15 分钟滚动扫描未来 7 天；BullMQ 延迟提醒、确定性 jobId 去重、指数退避和失败保留。
- M2 幂等：任务完成/跳过/撤销支持 24 小时 `Idempotency-Key` 预留、请求摘要冲突检测和原响应重放。
- M2 通知日志：Worker 写入 QUEUED/SENT/DELIVERED/FAILED，API 支持按状态查询；管理员可重试发送失败任务，Receipt 失败会使用最新有效设备 Token 重建发送任务。
- M2 生命周期修复：撤销后的任务可再次完成并复用/恢复原任务记录，不再触发唯一键冲突。
- M2 App 计划管理：计划列表、编辑、暂停与恢复交互已接入真实 API。
- M2 Expo 推送：设备权限由用户主动开启，注册 Development Build 的 Expo Push Token，并支持设备会话级 Token 管理。
- M2 推送 Token 生命周期：退出当前设备、退出全部设备、远程撤销会话及 Refresh Token 重放会原子停用关联 Token；任务生成和失败重试还会二次要求会话未撤销且未过期，避免退出后继续收到家庭提醒。
- M2 家庭提醒边界：成员被移除或主动离开时，会清空其计划及未完成任务负责人；任务生成只向当前有效家庭成员发送，失败通知重试也会拒绝向已离开的成员重新发送。
- M2 推送点击闭环：前台、后台与冷启动通知响应统一校验任务/家庭标识，只允许切换到当前账号所属家庭并打开真实任务详情；伪造路径和跨家庭 payload 会被拒绝。
- M2 飞书通知：家庭管理员可配置、测试和移除飞书机器人；Webhook 使用 AES-GCM 加密保存，接口只返回脱敏信息。
- M2 多通道发送：Worker 支持 Expo Push、飞书和开发通道，按接收人生成确定性任务并统一记录发送结果与重试状态。
- M2 Expo 回执闭环：推送票据提交后延迟查询 Expo Receipt，区分 SENT 与 DELIVERED；回执查询采用指数退避，永久失败写回安全错误，`DeviceNotRegistered` 会自动停用对应设备 Token。
- M2 轻量离线：App 使用 SQLite WAL 持久化任务完成、跳过和撤销操作，保留原始请求体与幂等键，联网后自动重放并区分冲突/失败。
- M3 记录 API 第一阶段：饮食、饮水、体重、排便、呕吐、用药、疫苗、驱虫和铲屎的类型专属校验；创建幂等、时间线筛选、详情、乐观锁编辑、软删除与 30 天管理员恢复。
- M3 记录权限：所有手动记录强制校验猫咪属于当前家庭；成员仅可编辑/删除自己的记录；任务生成记录禁止直接改写。
- M3 App 第一阶段：快速新增已开放 7 类日常记录，支持选择猫咪、异常标记、类型专属字段与备注；记录 Tab 接入真实时间线和按猫筛选。
- M3 App 记录详情：时间线条目可进入详情，展示类型专属字段；手动记录支持备注、异常标记编辑和软删除，任务生成记录明确只读及正确修改路径。
- M3 记录离线：手动新增使用稳定 `clientId` 与幂等键写入 SQLite 操作队列；时间线缓存最近 90 天，断网时回退本机数据，恢复联网后自动重放。
- M3 初步洞察：记录时间线展示近期异常数量和最近 7 次体重变化，数据不足时提供明确引导。
- M3 健康事件模型：新增观察中/已恢复状态、开始与恢复时间、摘要、创建人、乐观锁，以及症状/观察/治疗三类记录关联。
- M3 健康事件 API：列表、创建、详情、编辑、关联/移除记录和恢复；创建幂等，跨家庭/跨猫关联受阻，成员仅能修改自己的事件。
- M3 健康事件 App：异常记录可建立健康事件；支持观察中/已恢复列表、事件详情、关联记录回看和标记恢复，并明确非诊断用途。
- M3 医疗档案模型：新增疫苗、驱虫、用药三类结构化档案，包含品牌、批次、剂量、机构、下次日期、反应、备注和乐观锁。
- M3 医疗档案 API：查询、创建、详情、编辑和软删除；仅家庭管理员可写，日期范围、猫咪归属、版本冲突和删除操作受校验与审计保护。
- M3 就医摘要：支持 JSON 与可打印 HTML，聚合猫咪资料、医疗档案、异常记录和健康事件，并固定展示非诊断免责声明。
- M3 医疗档案 App：记录中心可进入医疗档案列表，管理员可新增疫苗、驱虫和用药事实，普通成员保持只读。
- M3 PDF 就医摘要：服务端使用内置思源黑体子集生成 A4 PDF，包含猫咪资料、医疗档案、健康事件、异常记录、免责声明、页码和下载文件名。
- M3 摘要原始响应：HTML/PDF 绕过 JSON 包装，返回正确可直接打开的正文与 MIME；JSON 继续遵守统一 API envelope。
- M3 医疗档案操作闭环：App 支持按猫筛选、详情、全部结构化字段编辑、软删除；普通成员详情保持只读，管理员操作前显示后果说明。
- M3 医疗日期：新增和编辑均支持 `YYYY-MM-DD` 发生日期与下次日期，客户端校验真实日期和先后关系；当天发生时间使用当前时刻，避免时区导致未来时间误判。
- M3 App 摘要分享：按当前猫咪带认证下载 PDF 到 App 缓存，再调用 iOS/Android 系统分享；文件名安全化，不暴露长期下载地址。
- M3 同步冲突中心：SQLite 持久化冲突/失败操作，集中展示本机请求、服务端任务状态、版本、时间和错误原因；支持保留服务端或基于最新版本重试。
- M3 冲突安全：已完成/已跳过等终态默认不提供覆盖重试；仅服务端仍为待办的版本冲突可更新版本后重放，医疗确认字段不在诊断界面明文展示。
- M3 冲突入口：任务与记录页面会持续统计既有冲突，不因重新进入页面而丢失提示，可直接进入处理中心。
- M3 健康事件操作闭环：事件创建人或家庭管理员可编辑标题与摘要，按症状/观察/治疗继续关联同一只猫的有效记录，并可解除错误关联而不删除原记录。
- M3 健康事件选择器：自动排除已关联记录和其他猫咪记录，展示记录类型、异常状态与发生时间；关联完成后返回详情自动刷新。
- M3 健康事件只读：非创建人的普通成员可查看完整事件和关联记录，但不能修改、恢复或解除关联。
