- M3 猫咪档案聚合：新增单猫档案概览 API 与体重趋势 API，聚合照顾计划/待办/逾期、体重、医疗档案、健康事件、最近记录和最近照片；App 猫咪详情页已接入聚合概览，并保留管理员编辑、成员只读和相册/医疗档案真实跳转。
- M3 App 猫咪档案返回保护：编辑猫咪基础资料时，返回、Android 返回键、记录/相册/医疗档案快捷入口和最近记录/照片跳转都会在未保存修改时提示离开风险。
- M3 公共猫砂盆记录：手动铲屎记录支持“公共猫砂盆”归属，列表与详情明确展示公共归属；非铲屎记录仍禁止无猫咪提交。
- M3 单猫上下文记录：新增记录页支持 `petId`/`pet` 来源参数，猫咪档案可直接发起记录；提交前会再次显示本次记录归属。
- M3 单猫上下文相册：猫咪档案进入相册会保留单猫筛选，继续上传照片时默认绑定来源猫咪，上传成功后返回同一只猫的相册筛选；无效参数安全回退为全部相册。
- M3 照片记录闭环：照片上传成功后会生成 `PHOTO` 时间线记录；服务端校验照片属于当前家庭且已绑定记录猫咪，并写入照片与记录关联表，记录列表和详情使用照片中文摘要。

- M3 照片记录重试闭环：照片文件已上传成功但时间线记录生成失败时，可再次提交并复用已上传照片生成 `PHOTO` 记录，不再误判为照片上传失败。
- M3 App 照片上传返回保护：照片上传页关闭、取消和 Android 返回键会在已选照片、备注或猫咪归属变更时提示未保存离开风险。
- M3 App 照片上传归属加固：照片上传页将猫咪归属加载中、加载失败、家庭无猫、未选择照片和未绑定猫咪拆成明确状态；上传中锁定归属、备注、相册和拍照入口，避免照片与时间线记录归属不一致。
- M3 App 照片详情返回保护：照片详情编辑备注或绑定猫咪后，返回、Android 返回键和 iOS 侧滑返回会提示未保存离开风险或被禁用；保存/删除等忙碌状态会阻止继续修改绑定，避免状态不一致。
- M3 App 相册网格加固：相册列表改为按屏幕宽度和设计 token 计算双列/通栏照片尺寸，覆盖 360/375/390/420px，避免百分比宽度叠加 gap 在小屏或安卓字体缩放下产生不稳定换行。
- M3 App 照片数量上限：照片上传页明确最多 9 张，达到上限后相册和拍照入口进入禁用态并显示可恢复说明。
- 数据安全补强：猫咪删除会在同一事务内停用关联计划、取消未完成任务并写入审计；任务生成器不会再处理已删除猫咪的计划。
- 记录权限补强：任务生成记录禁止直接改写；医疗记录仅管理员可写；普通成员只能修改自己创建的非医疗手动记录，App 与 API 使用同一权限边界。
- 生日日期按家庭 IANA 时区校验真实日历日期和“不得晚于今天”，避免 UTC/中国时区边界误判。
